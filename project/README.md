# Проект по информационному поиску: BM25, word2vec, Navec
Авторы: Майя Горшенина, Соня Черноусенко

## Описание проекта

В проекте реализован поиск по корпусу русскоязычных текстов с использованием трех вариантов индексации:

- **BM25**
- **word2vec**
- **Navec**

На вход подается текстовый запрос и тип индекса.  
На выходе программа возвращает **top-k наиболее подходящих документов** и выводит **время, затраченное на поиск**.

Поиск запускается через **CLI** на базе `argparse`.

---

## Реализованные индексы

### BM25

BM25 реализован вручную в файле `indexing/bm25_matrix.py`.

Особенности реализации:

- используется матричное представление
- строится TF-матрица
- заранее вычисляется IDF
- для поиска используется стандартная формула BM25
- повторы слов в запросе не учитываются отдельно

**Оценка сходства:** BM25 score.

BM25 в этом проекте не является обучаемой моделью.  
Это классический алгоритм ранжирования для информационного поиска.

---

### word2vec

`word2vec` реализован в файле `indexing/word2vec_index.py`.

В проекте используется модель, **обученная на предобработанном корпусе проекта**, хранящамся в файле `data_short_pp`.  
Обучение выполняется скриптом `models/train_word2vec.py`.

Параметры обучения:

- `vector_size=100`
- `window=5`
- `min_count=1`
- `sg=1` - skip-gram
- `epochs=20`

После обучения модель сохраняется в:

    project/models/word2vec/model.kv

Как считается вектор документа и запроса:

- текст разбивается на токены
- для каждого токена берется эмбеддинг
- вектор документа - среднее арифметическое векторов токенов
- вектор запроса считается аналогично

**Оценка сходства:** косинусное сходство.

---

### Navec

`Navec` реализован в файле `indexing/navec_index.py`.

В проекте используется **предобученная статическая модель эмбеддингов Navec**, которая загружается из файла:

    project/models/navec/model.tar

**Источник модели:** https://github.com/natasha/navec, `navec_hudlit_v1_12B_500K_300d_100q.tar`

Как считается вектор документа и запроса:

- для каждого токена берется вектор из Navec
- вектор документа - среднее арифметическое векторов токенов
- вектор запроса считается аналогично

**Оценка сходства:** косинусное сходство.

## Корпус

В проекте используется корпус русскоязычных текстов рецептов https://huggingface.co/datasets/ScoutieAutoML/recipes_for_dishes_and_food_with_vectors_sentiment_ners?library=pandas

Файлы данных:

- `data/data.csv` - полный датасет
- `data/data_short.csv` - уменьшенная выборка
- `data/data_short_pp.csv` - уменьшенная выборка после предобработки

Загрузка и подготовка данных находятся в файле `data/loader.py`.

## Назначение основных файлов

### `app_cli.py`

Главный файл CLI.

Что делает:

- читает аргументы командной строки
- загружает данные
- при необходимости загружает или создает предобработанный корпус
- строит выбранный индекс
- запускает поиск
- выводит top-k результатов и время поиска

### `data/loader.py`

Загрузка корпуса и формирование уменьшенной выборки.

### `utils/preprocessing.py`

Вся логика предобработки текста:

- приведение к нижнему регистру
- замена `ё` на `е`
- удаление лишних пробелов и переносов строк
- токенизация регулярным выражением
- лемматизация с помощью `pymorphy3`
- удаление русских стоп-слов из `nltk`

### `utils/input_validation.py`

Проверка входных данных.

В частности, проверяется, что запрос написан на русском языке.

### `indexing/base.py`

Общие вспомогательные функции для индексов:

- работа со словарем токенов
- построение TF-матрицы
- предобработка запроса
- выбор top-k документов

### `indexing/bm25_matrix.py`

Реализация индекса BM25.

### `indexing/semantic_base.py`

Общая база для семантических индексов:

- построение dense-векторов документов
- косинусное сходство
- общий поиск для word2vec и Navec

### `indexing/word2vec_index.py`

Реализация индекса на основе word2vec.

### `indexing/navec_index.py`

Реализация индекса на основе Navec.

### `models/train_word2vec.py`

Скрипт обучения word2vec на предобработанном корпусе.

## Использование CLI

### Формат команды

    python project/app_cli.py --query "<текст запроса>" --index <тип_индекса> --top-k <число>

### Параметры

#### `--query`

Текст поискового запроса.

Пример:

    --query "борщ со сметаной"

#### `--index`

Тип индекса.

Допустимые значения:

- `bm25`
- `word2vec`
- `navec`

#### `--top-k`

Сколько документов вывести.

Пример:

    --top-k 5

---

## Примеры запуска

    python project/app_cli.py --query "борщ со сметаной" --index bm25 --top-k 5
    python project/app_cli.py --query "куриный суп" --index word2vec --top-k 5
    python project/app_cli.py --query "полезные сладости" --index navec --top-k 10

---

## Что выводит программа

После запуска программа выводит:

- выбранный индекс
- время поиска в секундах
- список найденных документов

Формат результата:

- номер результата
- `doc_id`
- `score`
- фрагмент текста документа

## Особенности реализации

- запрос проходит ту же предобработку, что и документы
- для семантических моделей вектор документа считается как среднее по векторам слов
- если ни один токен запроса не найден в модели, результат может быть пустым
- если `top-k <= 0`, результат также будет пустым
- время поиска измеряется через `time.perf_counter()`
